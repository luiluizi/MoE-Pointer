# 遗传算法 vs 模拟退火算法性能分析

## 问题描述
在默认参数设置下，两种算法的求解时间相近，但遗传算法的求解效果远差于模拟退火。

## 关键差异分析

### 1. 适应度函数设计问题（**最可能的原因**）

#### 模拟退火
```python
return total_value - total_cost - total_penalty  # 线性函数
```

#### 遗传算法
```python
return math.exp((total_value - total_cost - total_penalty) / max(sum(self.value), 1) * 10)
```

**问题分析**：
- **指数函数导致适应度差异被放大**：即使目标函数值差异很小，经过指数变换后适应度差异会变得非常大
- **选择压力过大**：轮盘赌选择时，适应度高的个体被选中的概率会远高于其他个体，导致种群过早收敛
- **缩放因子问题**：`/ max(sum(self.value), 1) * 10` 这个缩放可能导致适应度值过大或过小，影响选择效果

**影响**：
- 种群多样性快速丧失
- 容易陷入局部最优
- 搜索空间探索不足

### 2. 交叉操作设计缺陷

#### 问题1：交换后续所有阶段
```python
def exchange(self, parent1, parent2, child1, child2, m, stage_id, pre_assign=-1):
    self.exchange_one_stage(parent1, parent2, child1, child2, m, stage_id, pre_assign)
    for i in range(stage_id + 1, 6):  # 交换后续所有阶段
        self.exchange_one_stage(parent1, parent2, child1, child2, m, stage_id=i, pre_assign=-1)
```

**问题**：
- 只交换一个请求的分配，但会交换该请求的所有后续阶段
- 这可能导致解的不一致性，因为：
  - 如果只交换了stage1，但stage2和stage3没有相应调整，可能违反阶段顺序约束
  - 父代1的stage1可能和父代2的stage2/3不匹配

#### 问题2：交叉操作过于保守
- 只交换一个请求的分配，搜索空间探索有限
- 如果交叉失败（不满足约束），直接返回父代，没有产生新解

### 3. 种群更新策略问题

#### 当前实现
```python
# 更新种群
population = new_population  # 完全替换，没有精英保留
```

**问题**：
- **没有精英保留机制**：最佳解可能在新一代中丢失
- **没有多样性保持**：种群可能快速收敛到局部最优
- 虽然记录了 `best_solution`，但不在种群中，无法参与后续进化

**对比模拟退火**：
- 始终保留历史最佳解
- 通过温度机制平衡探索和利用

### 4. 搜索强度差异

#### 模拟退火
- 每次迭代：生成1个邻域解，评估1次
- 总评估次数：5000次（max_iter=5000）
- 每次都是有效搜索（即使不接受，也探索了邻域）

#### 遗传算法
- 每代：生成 population_size 个新解
- 总代数：1000代（max_generations=1000）
- 但很多新解可能不满足约束，被丢弃或返回父代

**实际有效搜索次数**：
- 交叉成功率未知（可能较低）
- 变异成功率：最多5次重试，可能很多失败
- 实际有效的新解可能远少于理论值

### 5. 选择机制问题

#### 轮盘赌选择
```python
normalized_fitness = np.array(fitness).clip(min=1e-5)
normalized_fitness = normalized_fitness / normalized_fitness.sum()
parent1_id, parent2_id = self.rng.choice(..., p=normalized_fitness, size=2)
```

**问题**：
- 如果适应度函数是指数函数，适应度差异会被放大
- 可能导致某些个体被过度选择，某些个体几乎不被选择
- 没有考虑多样性

### 6. 参数设置问题

#### 默认参数对比
- **模拟退火**：`max_iter=5000`, `cooling_rate=0.99`
- **遗传算法**：`population_size=10`, `max_generations=1000`, `mutation_rate=0.3`, `crossover_rate=0.5`

**问题**：
- 种群大小太小（10）：多样性不足
- 变异率可能偏低（0.3）：探索不足
- 交叉率0.5：一半时间不进行交叉，搜索效率低

### 7. 初始解质量

两者使用相同的 `initial_solution()` 方法，初始解质量相同。

但遗传算法从10个相同的初始解开始（如果随机种子相同），多样性为0。

## 根本原因总结

### 主要原因（按重要性排序）

1. **适应度函数设计不当**（最重要）⭐⭐⭐⭐⭐
   - **指数函数放大差异**：假设两个解的目标函数值分别为 100 和 110（差异10%）
     - 模拟退火：直接使用 100 vs 110
     - 遗传算法：`exp(100/scale*10) ≈ 2.7e43` vs `exp(110/scale*10) ≈ 2.2e48`（差异巨大）
   - **选择压力过大**：轮盘赌选择时，适应度高的个体几乎总是被选中
   - **种群快速收敛**：几代后种群就变得非常相似，失去多样性
   - **数值稳定性问题**：指数函数可能导致数值溢出或精度问题

2. **没有精英保留机制**⭐⭐⭐⭐
   - **最佳解可能丢失**：如果最佳解在新一代中没有被选中，就会丢失
   - **无法保证单调性**：解的质量可能下降
   - **对比模拟退火**：始终保留历史最佳解，保证单调递增

3. **交叉操作设计缺陷**⭐⭐⭐⭐
   - **交换所有后续阶段**：`exchange` 方法会交换从 `stage_id+1` 到 6 的所有阶段
     - 例如：交换 stage 0 (pickup1) 时，会同时交换 stage 1-5
     - 这可能导致解的不一致性，因为父代1的stage1和父代2的stage2/3可能不匹配
   - **交叉失败率高**：由于交换了多个阶段，更容易违反约束
   - **实际搜索效率低**：很多交叉操作失败，返回父代，没有产生新解

4. **种群多样性不足**⭐⭐⭐
   - **种群大小太小**：只有10个个体，多样性有限
   - **初始种群可能相同**：如果随机种子相同，10个初始解可能完全相同
   - **没有多样性保持机制**：没有拥挤距离、共享适应度等方法

5. **搜索策略差异**⭐⭐⭐
   - **模拟退火**：每次迭代都是有效搜索（即使不接受，也探索了邻域）
   - **遗传算法**：很多操作可能失败（交叉、变异），实际有效搜索次数少
   - **评估次数对比**：
     - 模拟退火：5000次评估（每次都是有效搜索）
     - 遗传算法：理论10000次（10个体×1000代），但实际有效次数可能远少于此

### 5. 改进选择机制
```python
# 方案1：锦标赛选择（推荐，减少对适应度绝对值的依赖）
def tournament_selection(self, population, fitness, tournament_size=3):
    selected = []
    for _ in range(2):  # 选择2个父代
        tournament_indices = self.rng.choice(len(population), tournament_size, replace=False)
        tournament_fitness = [fitness[i] for i in tournament_indices]
        winner_idx = tournament_indices[np.argmax(tournament_fitness)]
        selected.append(population[winner_idx])
    return selected

# 方案2：排名选择（不依赖适应度绝对值）
def rank_selection(self, population, fitness):
    # 基于排名而非适应度值进行选择
    ranks = np.argsort(np.argsort(fitness))  # 转换为排名 [0, 1, 2, ...]
    # 排名越高，选择概率越大
    selection_probs = (ranks + 1) / (ranks + 1).sum()
    parent1_id, parent2_id = self.rng.choice(len(population), p=selection_probs, size=2)
    return population[parent1_id], population[parent2_id]
```

## 最关键的三个问题（必须优先解决）

### 1. 适应度函数（指数函数）⭐⭐⭐⭐⭐
**影响**：导致选择压力过大，种群快速收敛到局部最优
**解决方案**：改为线性缩放或直接使用目标函数值，配合排名选择

### 2. 没有精英保留 ⭐⭐⭐⭐
**影响**：最佳解可能丢失，无法保证解的质量单调递增
**解决方案**：每代保留最佳个体（elite保留）

### 3. 交叉操作交换所有后续阶段 ⭐⭐⭐⭐
**影响**：导致解不一致，交叉失败率高
**解决方案**：只交换当前阶段，不交换后续阶段
